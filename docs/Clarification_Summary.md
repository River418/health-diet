## 一、已澄清问题清单

### P0-1: D3 - 内容接入技术方案 ✅

**原问题**: 微信视频号/公众号内容接入技术方案不明确

**Judy(Dev)澄清结果**:
- **推荐方案**: 选项B（自建内容库）+ 选项A（申请平台API权限作为补充）
- **理由**:
  1. 自建内容库可确保内容版权合规，避免爬虫带来的法律风险
  2. 申请微信/抖音官方内容API需要企业资质审核，周期约2-4周
  3. 核心食疗配方内容建议自建，第三方内容作为信息流补充
- **技术实现要点**:
  - 自建CMS系统管理核心配方内容
  - 申请微信内容开放平台API权限
  - 抖音开放平台内容接入申请
  - 建立内容审核机制（人工+AI审核）

### P0-2: UI2 - 异常状态设计 ✅

**原问题**: 缺少加载状态、空状态、错误状态的设计说明

**Linda(Designer)澄清结果**:

| 状态类型 | 设计规范 |
|---------|---------|
| **加载状态** | 使用骨架屏(Skeleton) + 品牌色(#4CAF50)渐变动画，避免使用旋转Loading图标 |
| **空状态** | 居中展示插画+文案，插画风格：温暖手绘风，文案示例："暂无相关配方，换个关键词试试~" |
| **错误状态** | 分三级：①网络错误（重试按钮）②服务器错误（提示+客服入口）③内容不存在（404页面） |
| **全局规范** | 所有异常状态页面保持顶部导航栏，底部TabBar常驻，确保用户可随时退出 |

**补充说明**:
- 骨架屏高度与实际内容保持一致，避免跳动
- 空状态插画使用Lottie动画，增加亲和力
- 错误状态按钮使用主色调，突出可操作建议

### P0-3: Q2 - 登录异常场景验收标准 ✅

**原问题**: 缺少异常场景验收标准

**Alex(QA)澄清结果**:

| 异常场景 | 验收标准 |
|---------|---------|
| **网络中断** | 检测到网络异常时，显示"网络连接失败，请检查网络设置"，提供"重试"按钮，点击后重新发起登录请求 |
| **用户拒绝授权** | 用户点击"拒绝"后，显示友好提示"授权后可享受更多功能"，提供"重新授权"按钮，不强制要求授权 |
| **Code失效** | 登录code过期(5分钟)后，自动重新获取code并发起登录，用户无感知，后台记录重试次数 |
| **Token过期** | 自动刷新token，刷新失败时引导用户重新登录，保留用户已填写数据 |
| **多端登录冲突** | 同一账号在新设备登录时，原设备收到提示"账号已在其他设备登录"，不强制下线 |

**Sabrina(UO)补充用户体验流程**:
- 所有异常提示使用Toast + 图标形式，3秒自动消失
- 网络恢复后自动重试之前的操作
- 提供"帮助与反馈"入口，引导用户解决问题

---

### P0-4: Q8 - 合规性测试标准 ✅

**原问题**: 缺少合规性测试标准（免责声明验证方法）

**Alex(QA) + Linda(Designer)联合澄清结果**:

**验证方法**:
1. **自动化测试**: 遍历所有配方详情页，检查DOM中是否包含免责声明元素
2. **视觉验证**: 截图对比，确保免责声明在首屏可见区域（无需滚动）
3. **多设备验证**: 覆盖iPhone SE(小屏)到iPhone 15 Pro Max(大屏)

**通过标准**:
- 100%配方页面展示免责声明（抽样100个页面，通过率100%）
- 免责声明在页面加载后3秒内可见
- 用户可收起/展开，但默认状态为展开
- 收起后显示提示图标，点击可再次展开

**Linda提供视觉规范**:
- 位置：页面顶部，标题下方
- 样式：浅黄色背景(#FFF9E6) + 警示图标 + 12px字号
- 高度：展开状态60px，收起状态30px
- 文案："本内容仅供参考，不能替代专业医疗建议。如有疾病请及时就医。"
