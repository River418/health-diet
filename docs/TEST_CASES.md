## 1. 测试用例概述

### 1.1 测试用例分级

| 优先级 | 覆盖范围 | 测试重点 |
|-------|---------|---------|
| **P0** | 用户系统、配方查询、收藏功能 | 核心功能，必须100%通过 |
| **P1** | 评论系统、内容信息流 | 重要功能，影响用户体验 |
| **P2** | 消息队列、异常场景 | 增强功能，边界场景 |

### 1.2 测试用例编号规则

```
HD-TC-[模块]-[子模块]-[序号]
```

**模块代码**:
- USR: 用户系统 (User System)
- RCP: 配方系统 (Recipe System)
- FAV: 收藏功能 (Favorite)
- CMT: 评论系统 (Comment)
- CNT: 内容系统 (Content)
- SCH: 搜索功能 (Search)
- KNG: 金刚位系统 (KingKong)
- COM: 合规提示 (Compliance)
- MQ: 消息队列 (Message Queue)
- EXC: 异常场景 (Exception)
